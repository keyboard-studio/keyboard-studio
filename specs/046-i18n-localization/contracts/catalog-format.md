# Contract — message-catalog format & id namespace

The interface between the studio source, the committed catalogs, and Crowdin.

## Catalog file

- **Format**: minimal JSON, flat `{ "<id>": "<text>" }`. No nesting, no metadata.
- **Encoding**: UTF-8, LF, keys in extractor order (order is not significant — the
  drift gate compares order-independently).
- **Source (`en`)**: value = English source text.
- **Target (`<lang>`)**: value = translation, or `""` if untranslated.
- **Generated by**: `pnpm --filter @keyboard-studio/studio messages:extract`
  (Lingui). Never hand-author the file structure; only translations may be
  hand-edited in target files (Crowdin owns them in practice).

## Message id namespace

- **Grammar**: `area ( "." segment )+`, lowercase, dot-separated.
- **`area`** = a UI region: `welcome`, `nav`, `output`, and (as the sweep
  proceeds) `survey`, `editor`, `dashboard`, `ui`, …
- **Examples**: `welcome.title`, `nav.language`, `output.submit.button.submit`,
  `output.download.aria.ready`.
- **Stability rule**: an id is a permanent handle. Changing English text keeps
  the id (that is the whole point). Renaming an id is a deliberate act that
  orphans its translations — allowed only when the string's *meaning* changed.

## Authoring rules

- JSX text → `<Trans id="…">English</Trans>` (`@lingui/react/macro`). Interpolate
  variables inside `Trans`; never concatenate translated fragments.
- Attribute / non-JSX strings → the `t` macro (prefer `const { t } = useLingui()`
  from `@lingui/react/macro` for context-bound reactivity).
- Counts → the ICU `plural` macro, not `n === 1 ? … : …`.
- Do **not** wrap: code identifiers, test ids, technical tokens/paths, format
  examples (e.g. `you@example.com`), or values that are already data (echoed ids,
  dynamic error strings from shared pipelines).
- Language autonyms in a language picker are **not** translated.

## Fallback contract

Runtime resolves active-locale → English source (via the id) → never the raw id,
never blank. A missing whole-locale catalog falls back to `en`.
