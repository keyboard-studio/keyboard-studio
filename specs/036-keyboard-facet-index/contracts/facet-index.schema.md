# Contract: Facet Index Artifact Schema

Governs `docs/keyboard-facet-index.json`. Machine-generated by `utilities/facet-index`; validated at
build time (loud failure, FR-008) and again by `utilities/facet-index-lint` in `pnpm lint` (D7). JSON
Schema below; the lint additionally enforces the cross-checks.

## Cross-checks (the lint enforces beyond shape)

- **X1 limits**: every categorization `value` (and every `distribution` key) is within its facet
  definition's `limits`. **The single most important check** — FR-008's out-of-limits build failure. (`open: true` facets skip the closed-set check but still validate shape.)
- **X2 distribution sums to ~1**: `histogram`/`enum` distributions sum to 1 ± ε **when `residue` is
  absent**; when `residue` is present (037: facets over a closed recognized-value keyspace, e.g.
  strategy-fingerprint), `distribution` + `residue` sums to 1 ± ε instead. Loud failure otherwise.
- **X3 full coverage (SC-001)**: for every keyboard in `keyboards`, `facets` has a key for every id in
  `manifest.facetIds`. A missing facet is a failure, not a silent gap.
- **X4 outcome↔tier consistency**: `analysisOutcome: fallback-only` ⇒ `provenanceTier ≠ content-derived`.
- **X5 manifest agreement**: `keyboardCount === |keyboards|`; `facetCoverage` tier counts sum to
  `keyboardCount` per facet; `unicodeVersion` matches the pin in `scripts/ucd-version.json`.
- **X6 determinism markers**: object keys sorted (keyboards by id, facets by id, distribution by key); no
  timestamp field inside the hashed payload. (FR-006)
- **X7 self-check**: the lint rejects a synthetic out-of-limits record and accepts a good one. (facet-lint
  F7 discipline)
