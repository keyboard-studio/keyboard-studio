# Phase 1 Data Model: Glottolog catalog + bridge

Types live in `packages/glottolog/src/types.ts` and are re-exported from the package root. `BaseKeyboard` is imported from `@keyboard-studio/contracts` ([`baseKeyboard.ts`](../../packages/contracts/src/baseKeyboard.ts)); no locked contract type is modified.

## Scalar aliases

```ts
export type Glottocode = string;   // stable Glottolog id, e.g. "stan1293"
export type Iso639P3 = string;     // ISO 639-3, lowercased, e.g. "eng"
export type Script = string;       // ISO 15924 subtag, e.g. "Latn"
```

## Entity: Languoid

A node in the Glottolog classification tree. This is the public, resolved shape returned by the catalog API.

| Field | Type | Notes |
|-------|------|-------|
| `glottocode` | `Glottocode` | Stable id (primary key). |
| `name` | `string` | Glottolog name. |
| `level` | `"family" \| "language" \| "dialect"` | FR-010; lets consumers judge dialect-level relatives. |
| `iso639p3` | `Iso639P3 \| undefined` | Present mainly at language level; absent for most families/dialects (edge case). |
| `parentId` | `Glottocode \| undefined` | Undefined ⇒ this languoid is a family root. |
| `familyId` | `Glottocode \| undefined` | Root of its family (self for a top-level family/isolate). |
| `isPseudoFamily` | `boolean` | True when `familyId` ∈ the curated pseudo-family set (FR-012); such languoids never register genealogical relatedness. |

**Relationships**: forms a tree via `parentId`. `ancestors()` returns the root-first path (D7) of enclosing groups (excluding self). `familyId` is a denormalised shortcut to the root for cheap same-family pre-filtering.

## Generated index (codegen output)

`packages/glottolog/src/generated/index.ts` — the checked-in slim index (D11). Records are compact; the resolved `Languoid`/`isPseudoFamily` are computed at load.

```ts
export interface LanguoidRecord {
  name: string;
  level: "family" | "language" | "dialect";
  iso639p3?: Iso639P3;
  parentId?: Glottocode;
  familyId?: Glottocode;
}

export const languoids: Readonly<Record<Glottocode, LanguoidRecord>>;
// Permissive ISO → glottocodes (D4). Arrays sorted; a code with one match has length 1.
export const byIso: Readonly<Record<Iso639P3, readonly Glottocode[]>>;
```

**Determinism**: object keys sorted; `byIso` arrays sorted; fixed record key order; file rewritten only on content change. Asserted by `codegen-determinism.test.ts`.

## Entity: RelatednessResult

Output of `relatedLanguages()` — one related languoid plus its closeness to the target.

| Field | Type | Notes |
|-------|------|-------|
| `languoid` | `Languoid` | The related languoid. |
| `sharedSubgroupDepth` | `number` | Depth of the deepest shared subgroup (D3); larger = closer. |
| `pathLength` | `number` | Total edges between target and this languoid; tie-breaker (D3). |

Ordering: `sharedSubgroupDepth` desc, then `pathLength` asc, then `glottocode` asc. No default cap (D9).

## Entity: KeyboardBaseCandidate

Output of the bridge (`findKeyboardBaseCandidates()`) — the actionable suggestion. One entry **per keyboard** (D10).

| Field | Type | Notes |
|-------|------|-------|
| `keyboardId` | `string` | The candidate base keyboard. |
| `tier` | `"genealogical" \| "script-fallback" \| "direct"` | `direct` = distance-0 (target has its own keyboard, FR-017); `genealogical` = same-family + same-script (Tier 1); `script-fallback` = same-script regardless of family (Tier 2, from existing `suggestBases`). |
| `script` | `Script` | The candidate's script — always equal to the target's script (FR-017b). |
| `closestRelative` | `{ iso639p3: Iso639P3; glottocode: Glottocode; distance: number } \| null` | The related language that ranked this keyboard; `null` for pure script-fallback with no genealogical link. Ranking key. |
| `alsoSupports` | `readonly Iso639P3[]` | Other target-relatives this keyboard supports (secondary metadata, D10). |
| `base` | `BaseKeyboard \| undefined` | The resolved base record when the caller supplied one (from `@keyboard-studio/contracts`). |

Ordering: `direct` first, then `genealogical` by closeness, then `script-fallback`; ties by `keyboardId`. Deduplicated per keyboard.

## Entity: Version pin / Source manifest

`scripts/glottolog-version.json` (checked in) and `packages/glottolog/data/glottolog/SOURCES.json` (generated by fetch).

| Field | Where | Notes |
|-------|-------|-------|
| `source` | pin | `https://github.com/glottolog/glottolog-cldf` |
| `commit` / `tag` | pin | Pinned release. |
| `urlTemplate` | pin | Raw-file URL with `{commit}` placeholder. |
| `path` | pin | `cldf/languages.csv`. |
| `sha256` | pin | Verified on fetch; placeholder fails loudly (FR-002). |
| `license` / `notice` | pin | CC-BY-4.0 + Glottolog attribution. |
| `bytes`, `recordCount`, `url` | manifest | Recorded provenance after fetch. |

## Entity: Curated pseudo-family set

`packages/glottolog/src/pseudo-families.ts` — a checked-in, version-pinned `readonly Set<Glottocode>` (D6). Any languoid whose `familyId` is in this set has `isPseudoFamily = true`. Reviewed at each pin bump (FR-005/FR-012).

## Validation rules (from requirements)

- `byIso` values are non-empty arrays; a missing ISO key ⇒ not-found (FR-008).
- `ancestors()` excludes self, is root-first, `[]` for isolates (FR-009, D7).
- `relatedLanguages()` excludes any languoid whose `isPseudoFamily` is true and excludes cross-family languoids (no shared subgroup) (FR-011/FR-012).
- Every `KeyboardBaseCandidate.script` equals the target script (FR-017b); candidates are unique by `keyboardId` (FR-016a).
- The bridge returns `[]` only when both tiers are empty (FR-015).
