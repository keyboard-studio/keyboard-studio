# Implementation Plan: Deterministic Facet Classifiers (Script + Two Representative Facets)

**Branch**: `037-facet-classifiers` | **Date**: 2026-07-14 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from [specs/037-facet-classifiers/spec.md](spec.md)

## Summary

Specify and implement the **classification system** and its first three classifiers — one per derivation
archetype — that compute the values [spec 036](../036-keyboard-facet-index/spec.md)'s facet index records.
036 owns the artifact shape, freshness, and lint; 037 owns the **algorithms**: a `script` classifier
(character-content archetype, the hardest — sets the evidentiary standard), a `strategy-fingerprint`
classifier (rule-structure archetype), and a `target-mix` classifier (declared-metadata archetype). Every
classifier is a pure, versioned, deterministic function `(analysis inputs, pinned reference data) →
Categorization` emitting exactly the 036 Entity 2 record, distinguishing confident / mixed / undetermined
and recording the provenance tier that produced the value.

**Technical approach** (grounded in current code — see [research.md](research.md)):
- The three classifiers live inside the existing `utilities/facet-index/` build tool (036 D1) as
  `classifiers/{script,strategy-fingerprint,target-mix}.ts`, each implementing one
  [Classifier contract](contracts/classifier.contract.md). No new package; out of `pnpm -r`.
- **Script** (US1): iterate `buildProducedSet(ir)` (NFC scalars, membership-only), map each to script via
  the pinned UCD slim lookup (`Scripts.txt` / `ScriptExtensions.txt` / `PropertyValueAliases.txt` /
  `Blocks.txt` from `lib/ucd/`, Unicode 17.0.0), excluding Common/Inherited from the denominator, giving
  **full weight** to every `Script_Extensions` member (the only monotonic scheme — shared chars strengthen,
  never dilute), normalizing by weighted total. Latin sub-profile (plain/extended/IPA) is a block-derived
  hint. Fallback chain: content → declared script subtags → `getLanguageDefaults().defaultScript` →
  undetermined.
- **Strategy fingerprint** (US2): `recognizePatterns(ir)`, then `distribution[S] = ownedRuleCount(S) /
  totalRules` with `residue = 1 − recognizedRatio` as a distinct field. Recognizer covers only S-01/S-02 of
  the S-01..S-13 catalog today — residue-dominated fingerprints are honest and expected (spec Assumption).
- **Target mix** (US3): union of `.kps <Targets>` (enum-validated, defaults to `windows`), `.kmn &TARGETS`
  (raw; `'any'` sentinel expands to all), and touch-layout artifact presence (`LAYOUTFILE` sibling) —
  artifact outranks declaration, mismatches flagged.
- **Determinism** (FR-001/SC-003): sorted keys everywhere, canonical-sorted `Script_Extensions` sets in
  codegen, no timestamps in the hashed payload, `0/0 → 0`. Fixtures are real corpus keyboards from the
  phonebook; a determinism test asserts byte-identical reruns.

**Corrections surfaced at planning depth** (folded into the design, flagged to km-doc as non-blocking):
`ImportStatus` has 4 members not 3 (`RoundTripDivergence`'s I2 check is a deferred stub pending Keyman Core
keystroke-runtime integration, not a kmcmplib/WASM limitation → 037's `analysisOutcome` is a deliberate
3-state subset); the strategy catalog is **S-01..S-13** (earlier drafts of this research/contract set
undercounted it as S-01..S-12, now corrected); `analyzedCoverage` is a genuinely new metric, and — for the
strategy-fingerprint classifier — means parse-opacity, distinct from the separate `residue` (recognizer-gap)
field; `&TARGETS` commonly holds the `'any'` sentinel.

## Technical Context

**Language/Version**: TypeScript (ESM), Node ≥ 20, pnpm 9. Classifiers run via `tsx` inside the
`utilities/facet-index/` standalone tool (not a workspace package — stays out of `pnpm -r`).

**Primary Dependencies** (all existing engine/contracts exports — no new runtime dep):
`parseKmn` (`packages/engine/src/index.ts:58` → `codec/parse.ts:674`),
`buildProducedSet` (`packages/contracts/src/ir/producedSet.ts:210`, contracts root),
`recognizePatterns` (`packages/engine/src/recognizer/index.ts:14`, engine root),
`parseKmnHeaderStores` (`packages/engine/src/compiler/parseKmnHeaderStores.ts:47`, engine root),
`parseKps` (`packages/engine/src/base-browser/kps-parser.ts:97`, deep relative import),
`getLanguageDefaults` (`packages/engine/src/langtags/index.ts:25`, `@keyboard-studio/engine/langtags`).
Pinned UCD slim lookup generated by the 036 `codegen-ucd.mjs` step.

**Storage**: No new storage. Classifiers emit into the 036 index (`docs/keyboard-facet-index.json`). Three
new content-owned facet definitions under `content/keyboard-facets/`: `script.yaml` (036 ships this),
`strategy-fingerprint.yaml`, `target-mix.yaml` (037 ships these two).

**Testing**: vitest per-classifier fixture tests inside `utilities/facet-index/classifiers/*.test.ts`
(clear-cut case per outcome from real corpus keyboards, FR-006); a determinism test (byte-identical rerun,
SC-003). The SC-001 hand-judged validation set (≥30 keyboards) is content-team judgment data.

**Target Platform**: Offline developer/CI machine (Node). No WASM, no network, no SPA. The studio only
*reads* the committed index (later wiring feature).

**Project Type**: Algorithms inside a standalone offline build utility (data-pipeline) + content-owned
facet definitions. Not a service, not UI.

**Performance Goals**: Classifiers add bounded per-keyboard cost to the 036 full run (target: full rebuild
over ~1,000 keyboards in one CI step). Script and strategy classifiers reuse the parse the 036 loop already
performs — no second parse.

**Constraints**: Deterministic (FR-001) — byte-identical output for identical source + pins + classifier
version; no timestamps/randomness/environment-dependent order. Offline (FR-005) — no WASM oracle (hence the
3-state `analysisOutcome` subset). Every value within its facet definition's `limits` or the build fails
(FR-008).

**Scale/Scope**: 3 classifiers × ~1,000 keyboards. Corpus scope = `../keyboards/release/**` (inherited from
036). Classifiers beyond the three are out of scope (they follow the standard set here).

**Structure Decision**: The classifiers are engine-owned code that slots into the **existing**
`utilities/facet-index/` tool built by 036 — 037 adds a `classifiers/` subdirectory implementing the shared
`Classifier` contract, one file per archetype, plus their fixture tests. No new package, no new tool. The
two additional facet **definitions** (`strategy-fingerprint`, `target-mix`) live in the content-owned
`content/keyboard-facets/` directory 036 established; `script.yaml` is shipped by 036. This keeps the
engine/content boundary (Article VI) crisp: engine ships algorithms under `utilities/`, content ships
definitions under `content/`, linked by `derivation.classifierId`.
