# Implementation Plan: Langtags-driven identity autocomplete

**Branch**: `km/030-langtags-identity-autocomplete` | **Date**: 2026-07-08 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/030-langtags-identity-autocomplete/spec.md`

## Summary

Redesign the opening identity questions of the survey so the author's language is **resolved from SIL langtags** instead of hand-typed. The first question becomes a free-text-with-suggestions English-name autocomplete that resolves a langtags entry; a region question fires only when the English name is ambiguous across regions; the own-language name is offered as a multi-choice picker (with free-text override) seeded from the resolved entry's local names; and the language code becomes an auto-filled confirmation step. This requires extending the langtags slim index (and the `LanguageDefaults`/`LanguageSummary` contract types) to retain the `names[]` / `localnames[]` / `regionname` data the current index collapses to one value each, then reworking the two identity flows (live IdentityLite `il_*` and the proposed, non-live full Phase A `language_name_*`) and their seeding logic. Governs [spec.md §8](../../spec.md) (workflow ordering) and builds on `specs/023-langtags-defaults` / `specs/002-defaults-engine`.

## Technical Context

**Language/Version**: TypeScript 5.x, Node ≥ 20, pnpm 9 (monorepo)

**Primary Dependencies**: React 18 + Vite (studio SPA); `@keyboard-studio/engine/langtags` (slim index generated by `scripts/codegen-langtags.mjs` from the pinned SIL `langtags.json`); `@keyboard-studio/contracts` (types + zod schemas); zod

**Storage**: N/A — langtags is a build-time fetched, SHA-pinned JSON (commit `99b856b`) codegen'd into a slim TS index; all survey state is in-memory (working-copy spine)

**Testing**: vitest (unit/integration: question-module fixtures, `flow-parity`, `loadModularFlow`, `buildStepGraph`, `stepHost` golden-walk); Playwright e2e (survey flow); langtags-index unit tests

**Target Platform**: browser SPA (studio) + Node library packages (engine, contracts)

**Project Type**: monorepo — engine + studio + contracts (Option 2-ish: library packages + frontend app)

**Performance Goals**: English-name typeahead feels instant (< 100 ms per keystroke) over the full langtags language set; no other perf-sensitive path

**Constraints**: reuse the already-pinned langtags commit (NO live fetch, NO second data source — FR-012); graceful degradation for languages absent from langtags (no dead ends — FR-003); do not touch the validator/300 ms debounce path; VirtualFS-only (no host-disk writes)

**Scale/Scope**: langtags carries a few thousand language tagsets; slim index is the derived lookup; two survey flows updated (one live, one proposed); ~4–5 question modules per flow plus one new region-disambiguation module
